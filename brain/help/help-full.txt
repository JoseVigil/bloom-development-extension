═══┫ USAGE ┣═══

  brain [GLOBAL_OPTIONS] <category> <command> [OPTIONS] [ARGS]

  • Global options (--json, --verbose) must precede <category>

  → View all commands: brain --help --full
  → Category help: brain <category> --help

═══┫ GLOBAL OPTIONS ┣═══

  • --json         Output in JSON format
  • --verbose      Enable detailed logging
  • --help         Show help information

=======================================================[ HEALTH ]=======================================================

                                          System health checks and diagnostics


  ▸ dev-check
    Check development environment and service status.
    brain health dev-check [OPTIONS]

    ▪ Options:
      -t, --timeout             Connection timeout per port check in seconds
      --json                    Output raw JSON
      -v, --verbose             Detailed logging of each port check

  ▸ full-stack
    Verify complete stack health (host, API, extension, Brain, onboarding)
    brain health full-stack [OPTIONS]

    ▪ Options:
      -t, --timeout             Timeout per check in seconds
      --json                    Output raw JSON
      -v, --verbose             Detailed logging

  ▸ native-ping
    Ping the native host to verify it's alive and responding.
    brain health native-ping [OPTIONS]

    ▪ Options:
      -t, --timeout             Timeout in milliseconds for ping response
      -p, --port                Specific TCP port to test (auto-detect if not provided)
      --check-ws/--no-check-ws  Also verify WebSocket port status

  ▸ websocket-status
    Verify WebSocket server status on localhost:4124
    brain health websocket-status [OPTIONS]

    ▪ Options:
      -t, --timeout             Connection timeout in seconds
      --test-sub                Test event subscription capability
      --json                    Output raw JSON
      -v, --verbose             Detailed connection logging

  ▪ ONBOARDING & SETUP
  ──────────────────────

  ▸ onboarding-status
    Check onboarding completion status (GitHub, Gemini, Nucleus, Projects)
    brain health onboarding-status [OPTIONS]

    ▪ Options:
      -r, --refresh             Force re-check, ignore cache
      --json                    Output raw JSON
      -v, --verbose             Detailed logging of each check

======================================================[ NUCLEUS ]=======================================================

                                        Nucleus project management and lifecycle


  ▪ CREATION & INITIALIZATION
  ─────────────────────────────

  ▸ create
    Initialize a new Bloom Nucleus V2.0 project structure.
    brain nucleus create [OPTIONS]

    ▪ Options:
      -o, --org <VALUE>         Organization name
      -p, --path                Target path (default: current dir)
      --output                  Output directory name (legacy, kept for compatibility)
      --url                     Organization URL
      -f, --force               Force overwrite existing directory

  ▪ EXPLORATION & DISCOVERY
  ───────────────────────────

  ▸ create-exp-intent
    Create a new exploration intent in the nucleus.
    brain nucleus create-exp-intent [OPTIONS]

    ▪ Options:
      -n, --name <VALUE>        Intent name (will be slugified)
      -p, --path                Nucleus path (default: current dir)
      --inquiry                 Initial inquiry question
      -d, --description         Intent description
      --projects                Comma-separated project names to include

  ▸ exp-discovery-turn
    Add a discovery turn to an exploration intent.
    brain nucleus exp-discovery-turn [OPTIONS]

    ▪ Options:
      -i, --intent <VALUE>      Intent ID or slug
      -p, --path                Nucleus path (default: current dir)
      --notes                   Turn notes/observations
      --analysis                Analysis summary for this turn

  ▸ exp-export-findings
    Export findings from an exploration intent.
    brain nucleus exp-export-findings [OPTIONS]

    ▪ Options:
      -i, --intent <VALUE>      Intent ID or slug
      -p, --path                Nucleus path (default: current dir)
      -f, --format              Export format (markdown, json)
      --include-raw             Include raw turn data

  ▸ delete
    Delete a complete nucleus including all its data.
    brain nucleus delete [OPTIONS]

    ▪ Options:
      -p, --nucleus-path <VALUE> Path to the Nucleus root directory to delete
      -f, --force               Skip confirmation prompt

  ▸ get
    Retrieve complete details of a nucleus.
    brain nucleus get [OPTIONS]

    ▪ Options:
      -p, --nucleus-path <VALUE> Path to the Nucleus root directory

  ▸ link
    Link an existing folder as a Nucleus project.
    brain nucleus link <PATH>

    ▪ Arguments:
      <PATH>               Path to existing project

  ▸ status
    Check if a directory is a valid Nucleus project.
    brain nucleus status [OPTIONS]

    ▪ Options:
      -p, --path                Nucleus path

  ▸ sync
    Synchronize projects and rebuild nucleus cache.
    brain nucleus sync [OPTIONS]

    ▪ Options:
      -p, --path                Nucleus path (default: current dir)
      --skip-git                Skip git pull operations
      -v, --verbose             Show detailed progress

  ▪ LISTING & INSPECTION
  ────────────────────────

  ▸ list
    List all Nucleus projects found in a parent directory.
    brain nucleus list [OPTIONS]

    ▪ Options:
      -d, --parent-dir          Parent directory to scan for nuclei (default: current directory)

  ▸ list-projects
    List all projects linked to the nucleus with their details.
    brain nucleus list-projects [OPTIONS]

    ▪ Options:
      -p, --path                Nucleus root path (default: current directory)
      -s, --strategy            Filter by project strategy (python, typescript, go, etc.)
      --active-only             Show only active projects

  ▪ ONBOARDING & SETUP
  ──────────────────────

  ▸ onboarding-complete
    Mark an onboarding step or entire onboarding as completed.
    brain nucleus onboarding-complete [OPTIONS]

    ▪ Options:
      -s, --step                Step name to mark as completed
      -a, --all                 Mark entire onboarding as completed
      -p, --path                Nucleus root path (default: current dir)

  ▸ onboarding-status
    Get the current onboarding status from nucleus configuration.
    brain nucleus onboarding-status [OPTIONS]

    ▪ Options:
      -p, --path                Nucleus root path (default: current dir)

  ▪ PROJECT OPERATIONS
  ──────────────────────

  ▸ project-info

    ▪ Arguments:
      <PROJECT_NAME>       Name of the project to inspect

    ▪ Options:

======================================================[ PROJECT ]=======================================================

                                           Project lifecycle and scaffolding


  ▪ ADDING & LINKING
  ────────────────────

  ▸ add
    Vincula un proyecto local existente a un Nucleus.
    brain project add <PROJECT_PATH> [OPTIONS]

    ▪ Arguments:
      <PROJECT_PATH>       Ruta al proyecto a vincular

    ▪ Options:
      -n, --nucleus <VALUE>     Ruta al Nucleus destino
      --name                    Nombre custom del proyecto (default: nombre de carpeta)
      --strategy                Forzar estrategia específica (default: auto-detect)
      --description             Descripción del proyecto
      --repo-url                URL del repositorio GitHub

  ▸ clone-and-add
    Clone a Git repository and automatically link it to the current Nucleus.
    brain project clone-and-add <REPO_URL> [OPTIONS]

    ▪ Arguments:
      <REPO_URL>           Git repository URL (HTTPS or SSH)

    ▪ Options:
      -d, --dest-path           Absolute path where to clone the repository. If not provided, clones to Nucleus root 
using repo name.
      -n, --nucleus-path        Path to the Nucleus project. Auto-detected from current directory if not provided.
      --name                    Custom name for the project (default: repository name)
      --strategy                Force specific technology strategy (e.g., python, typescript, android)
      --description             Project description

  ▸ detect
    Scan directory and detect project types automatically.
    brain project detect <PARENT_PATH> [OPTIONS]

    ▪ Arguments:
      <PARENT_PATH>        Parent directory to scan for projects

    ▪ Options:
      -d, --max-depth           Maximum depth for recursive scanning
      -s, --strategy            Filter projects by strategy type
      -c, --min-confidence      Filter by minimum confidence level (high, medium, low)

  ▸ load
    Validates and loads project information. 
    brain project load [OPTIONS]

    ▪ Options:
      -p, --path                Path to the project root

=====================================================[ FILESYSTEM ]=====================================================

                                         File operations and directory analysis


  ▸ compress
    Compress files into optimized JSON format for AI consumption.
    brain filesystem compress <PATHS> [OPTIONS]

    ▪ Arguments:
      <PATHS>              Paths to compress (files or directories)

    ▪ Options:
      -m, --mode                Compression mode: 'codebase' or 'docbase'
      -o, --output              Output directory (default: current directory)
      -e, --exclude             Patterns to exclude (can be used multiple times)

  ▸ extract
    Extract files from compressed JSON format.
    brain filesystem extract <JSON_FILE> [OPTIONS]

    ▪ Arguments:
      <JSON_FILE>          Compressed JSON file to extract

    ▪ Options:
      -o, --output              Output directory (default: extracted_<mode>)
      --verify-hashes/--no-verify-hashes Verify MD5 hashes during extraction

  ▸ tree
    Generate a visual directory tree structure with intelligent exclusions.
    brain filesystem tree <TARGETS> [OPTIONS]

    ▪ Arguments:
      <TARGETS>            Specific directories or files to include (supports absolute paths).

    ▪ Options:
      -o, --output              Output file path
      --hash                    Include MD5 hashes
      --export-json             Export JSON to disk
